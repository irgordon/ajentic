Trial package and evidence

A trial package collects local run information, a runbook, failure drill notes, evidence links, replay/status information, and boundary markers.

The runbook explains the manual steps an operator follows during a local trial workflow.

evidence capture records local observations and summaries so the operator can review what happened.

The trial execution harness shows local trial execution status and stop-condition observations. It does not expand provider execution.

Trial package and Evidence pages are local-only and non-public.

evidence is not authority. It supports review, but it is not release approval, not deployment approval, not public-use approval, and not production approval.
