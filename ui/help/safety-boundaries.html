Safety boundaries

AJENTIC local beta explains and displays local workflow state. It does not approve controlled human use.

  • No production approval. This is not production approval.
  • No release approval. This is not release approval.
  • No deployment approval. This is not deployment approval.
  • No public/general-use approval. This is not public-use approval.
  • No automatic provider trust. Provider output remains untrusted.
  • No arbitrary command execution.
  • No cloud/network provider path unless a future phase explicitly adds one.
  • No automated stop-condition enforcement.
  • No action authorization.
  • No replay repair.
  • No recovery promotion.

These boundaries protect the operator from mistaking local review information for authority.

This safety page is explanatory, local-only, and non-production.
