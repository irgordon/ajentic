Provider setup

Provider setup explains how AJENTIC connects to the local deterministic provider path.

Adapter contract

An adapter contract is a small description of the provider shape AJENTIC expects. It says which local provider path can be used and which fields are allowed.

Dry run

A dry run checks the adapter contract without treating the provider output as trusted. It helps find missing or unsafe setup before the workflow continues.

Constrained invocation

constrained invocation keeps provider use on the allowed local path.

Constrained invocation means AJENTIC uses only the allowed local provider path. AJENTIC does not run arbitrary commands.

Provider output remains untrusted. It is displayed for validation and review only.

This provider setup is local-only and non-production.
